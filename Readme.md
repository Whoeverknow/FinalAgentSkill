🔬 机制详解
它不是怎么工作的
大部分 AI 知识工具是这样工作的：你上传一堆 PDF → 系统切碎成几百个文本片段 → 存到向量数据库 → 你问问题时检索相关片段 → AI 拼出一个答案。
这叫 RAG（检索增强生成）。它每次都要重新发现知识，因为碎片之间没有持久关系。
它是怎么工作的
KM v3.1 走了完全相反的路：
你不是在做"搜索"
你是在建一个被 AI 持续维护的知识图谱
核心循环只有三步：
① 摄入：你说"保存这个"或上传一篇论文 → 一次性完整提取结构/数据/方法/结果/局限 → 写为草稿
② 编译：每天结束时审核草稿 → 校验格式和 GB 引文 → 检查矛盾 → 与已有知识建立双向链接 → 升级为正式条目
③ 生长：每个新条目不只是"被加入"——它会触发连锁更新：索引重排、标签图谱重建、概念图增加新节点、相关条目反向链接自动补上
与 RAG 的对比
RAGKM v3.1状态无状态——每次查询独立有状态——知识随时间累积每次查询时检索碎片 + 拼凑答案读摘要索引 → 定位条目 → 读完整条目关系每问一次重新发现预处理：交叉引用在摄入时就建立好了矛盾处理检测不到——矛盾碎片并排存在每周自动诊断，双方标注 + 交叉引用基础设施向量数据库 + 嵌入管线 + 检索逻辑一个文件夹，全是 .md 文件维护成本索引重建、嵌入更新LLM 在每次摄入时自动维护可追踪性碎片级引用（常有失真）每个声明精确溯源到源文件规模甜点百万级文档50-300 个源（个人/团队规模）
不是谁更高级——是完全不同的场景。RAG 解决"我有海量文档，每次都想搜"。KM v3.1 解决"我每周读几篇论文，知识需要积累和关联"。
与 LLM Wiki 的对比
KM v3.1 从 LLM Wiki 的同一个种子出发，但长出了一棵不同的树：
LLM Wiki 原版KM v3.1治理层CLAUDE.md 单文件三层分离：宪法 + 工作流 + 策略多窗口单会话假设三窗口并行：文本/图像/工具 → KNID 冲突解决上下文管理索引导航（隐式）形式化 token 预算（~8K 启动）+ SUMMARIES.md + CACHE.md知识生命周期永远增长Zipf 分级：hot→warm→cold→tombstone，引用频率驱动质量保障lint 扫描lint + 自动治疗（7 种病理各有 auto-fix 或 flag 策略）引用源链接GB/T 7714-2015 正式学术格式 + 引文残缺标注偏差控制隐式信任 LLM6 种偏差缓解（调和/确定性膨胀/新颖性/锚定/目的论重写/风格渗透）规则保护无对话污染屏障：闲聊不会意外修改 governance
LLM Wiki 是一个优雅的概念原型。KM v3.1 是这个原型经历了真实学术环境的多重压力后长成的生产系统。

📦 安装教程
你需要

一台电脑（Windows/Mac/Linux 都行）
Claude + Cowork（或任何支持文件读写的 LLM agent）
30 秒

三步启动
第一步：解压
把 km-v3.1-portable.zip 解压到任意位置。建议一个路径明确的文件夹，比如：
C:\Users\你的名字\KnowledgeBase\
或
~/Documents/knowledge-base/
解压后你会看到：
tools/     ← 管理文件（宪法/流程/策略/索引）
work/      ← 知识草稿区
knowledge/ ← 已发布知识库
conversations/ ← 会话记录
memory/    ← 持久化记忆
baseline/  ← 规则版本备份
README.md  ← 快速参考
第二步：打开对话
在 Claude/Cowork 中，把工作区文件夹指向你解压后的位置。
此时智能体会自动检测（它检查是否有 tools/CONSTITUTION.md 和 tools/CACHE.md）并宣布：

"知识管理模式已激活。当前共 1 条已发布条目。"

注意：不需要说任何特殊命令。不需要敲 /启动。它会自己检测到。
第三步：（可选）导入 Skill
如果这次对话中没有自动激活，导入 tools/SKILL-BOOTSTRAP.md 的内容。在 Cowork 中保存为 Skill，名称填 academic-knowledge-manager。
你还需要两个定时任务（可选但推荐）
这步需要手动创建（因为定时任务是绑定到 Cowork 实例的，不能打包）。打开 tools/SCHEDULED-TASKS.md 参考内容，创建：

daily-knowledge-review（每天凌晨 3 点）：自动扫描漏掉的知识、清理过期草稿
weekly-knowledge-analysis（每周六上午 10 点）：深度分析 + 6 种病理自动诊断和治疗

测试一下
对着智能体说：

"我想试试知识管理管线能正常工作吗，请给我看当前的 Hello World 条目"

它会读取 knowledge/2026/05.md 并返回那个格式参考条目。如果返回了——管线正常。
开始存第一条知识
说：

"保存这个"

然后随便说点你想记住的东西。第一条真实知识就会走完整管线：提取 → 审核 → 生成 KNID → 建立关联 → 写入 knowledge/ → 更新索引/标签/图谱。
关于 Hello World 条目
P0-IDEA-HELO-0000000001 只是一个格式示例。你可以：
选项 A：保留它，作为新条目的格式参考
选项 B：在存了第一条真实知识后删除它
删除方法：告诉智能体"删除 Hello World 条目"——它会从 knowledge/ 中移除并同步更新 INDEX.md / CACHE.md / SUMMARIES.md。
